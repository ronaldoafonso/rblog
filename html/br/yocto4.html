O que é o poky?

O poky é conhecido como uma distribuição de referência do projeto Yocto.

Ou melhor, ele é um template para a construção de um sistema Linux customizado.

Ele contém projetos como BitBake, OpenEmbedded Core e Metadata que nos guiam no processo de construção do sistema Linux customizado.

Eu vou descrever todos estes outros projetos em artigos futuros, mas por hora vamos ver como dar os primeiros passos com o poky.

Antes de mais nada, precisamos clonar o código fonte do poke e executá-lo em nossa build machine.

Basicamente devemos fazer o seguinte:

git clone http://git.yoctoproject.org/cgit/cgit.cgi/poky

cd pody

git fetch --tags

git checkout tags/yocto-3.2

Note que estou usando um branch específico aqui, ok?

Vamos usar o "default target" aqui. É o jeito mais fácil de se iniciar. Depois de você pegar a idéia de como o Yocto funciona, você poderá tentar targets mais avançados.

source oe-init-build-env

bitbake core-image-minimal

Agora você deve ter uma imagem que pode ser executada em seu computador. Claro, isso aconteçará através do QEMU, que é um emulador.

runqemu qemux86

Após alguns segundos você deve ver um dispositivo rodando como se fosse um dispositivo embarcado.
