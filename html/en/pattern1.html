What is the "Singleton" design pattern?

Well, it is considered the easiest design pattern, and the main idea behind it is that a class should have only one instance of it during the entire execution of a program.

The "singleton" design pattern is usually used when we don't need to have more than one instance of some class. For example, imagine that your application has to access a remote Linux server via an SSH connection and execute a command every five minutes. So, it doesn't make sense to open a new SSH connection each time a new command should be executed.

For situations like that, we can leverage the "singleton" design pattern.

I have two implementations of the "singleton" design pattern.

One in Python and another one in Golang.

If you look at the code, you'll see that, before returning a new instance, I check if another one hasn't already been created, if so, I just return that instance. Otherwise, I create a new instance and return it.
