Bueno, qué es poky?

Poky es conocido como una distribuición de referencia del proyecto Yocto.

Es decir, es un template para construir un sistema Linux personalizado.

Contiene proyectos como BitBake, OpenEmbedded Core y Metadata que nos guían en el proceso de construcción de un sistema Linux personalizado.

Voy a describir todos los otros proyectos uno por uno, pero aquí veamos cómo empezar con poky.

En primer lugar, necesitamos clonar el código poky y ejecutarlo en nuestra build machine.

Basicamente eso:

git clone http://git.yoctoproject.org/cgit/cgit.cgi/poky

cd pody

git fetch --tags

git checkout tags/yocto-3.2

Tenga en cuenta que estoy usando un branch específico aquí, ¿ok?

Usaré lo "default target" aquí. Es la forma más sencilla de empezar. Después de tener la idea básica de cómo lo Yocto funciona, puede probar otros más avanzados.

source oe-init-build-env

bitbake core-image-minimal

Ahora debes tener una imagen que puede ejecutar en su computadora. Por supuesto, lo ejecutará a través de QEMU, que es un emulador.

runqemu qemux86

Después de unos segundos, deberá ver un dispositivo funcionando como si fuera un dispositivo integrado.
