Ahora es el momento de hablar sobre comó configurar una build machine.

Lo qué es o para qué sirve una build machine?

Para crear una imagen de su sistema Linux personalizado, necesitará otro sistema Linux.

Bueno, necesita una distribución de Linux "tradicional" donde pueda descargar el proyecto Yocto, para que pueda construir un sistema Linux personalizado para usted.

Voy a usar un Debina 10 "buster" aquí, pero hay otras distribuciones de Linux que puede usar como build machine.

En el momento actual, los requisitos previos para una build machine son:

  • 50 Gbytes de espacio libre en disco.
  • Git 1.8.3.1 o mayor.
  • tar 1.28 o mayor.
  • Python 3.50 o mayor.
  • gcc 5.0 o mayor.

Dependiendo de su distribución Linux, necesitará una cierta lista de paquetes instalados.

En nel caso de Debian 10, eso será:

apt-get install gawk wget git-core diffstat unzip texinfo gcc-multilib build-essential chrpath socat cpio python3 python3-pip python3-pexpect xz-utils debianutils iputils-ping python3-git python3-jinja2 libegl1-mesa libsdl1.2-dev pylint3 xterm python3-subunit mesa-common-dev

Eso es todo!
