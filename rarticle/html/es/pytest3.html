Pytest - Artículo 3

Este es el tercero artículo sobre el framework para las pruebas Python más utilizado, "pytest".

Puedes encontrar el segundo aquí.

Hoy voy a hablar sobre "fixture".

Un "fixture" ès básicamente utilizado para inicializar sus pruebas.

Quizás te estás preguntando, como así?

Vamos a ver nostro projecto "rcalc".

Si lo clona y usa la etiqueta "v2.0", verá que creé el archivo "conftest.py".

Dentro de ese archivo podras ver un "fixture", o una función, que devuelve una secuenci de "tupes".

Estos "tuples" sirven como datos para nuestras pruebas.

Solo necesito definirlo en un solo lugar y puedo usarlo en cualquier prueba.

Eche un vistazo también al archivo "test_rcalc.py".

Allí se puede ver que acabo de pasar el "fixture" como parámetro para mis pruebas.

Y dentro de las pruebas puedo utilizar mi "data generator".

Ahora, cada vez que ejecute sus pruebas, "pytest" ejecutará el "fixture" por usted.

Otro ejemplo de "fixture" es la inicialización de la base de datos.

Bueno, eso es todo. Espero que estos artículos te hayan sido de ayuda.
