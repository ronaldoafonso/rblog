Pytest - Articolo 2

Questo è il secondo articolo sul framework piú utilizzato per i test nel mondo Python, il "pytest".

Puoi trovare il primo qui.

Nel primo articolo ho appena spiegato come installare "pytest".

Quindi, presumo che tu abbia già un ambiente virtuale con "pytest" installato.

L'argomento di questo articolo è la caratteristica "mark" di "pytest".

Lascia que ti chieda qualcosa.

Come puoi eseguire solo alcuni dei tuoi test invece di tutta la suite di test?

Nel caso lo avessi dimenticato, puoi trovare il codice del progetto "rcalc" qui.

Quindi, supponiamo di avere un modulo Python chiamato "rcalc.py" e di dover testare quattro operazioni di base di esso.

Di solito creiamo un file "test_rcalc.py" per testare il modulo "rcalc.py".

Dai! Farlo, clona il mio repository "rcalc", controlla il tag "v1.0" e dai un'occhiata ai file. Specialmente il file "test_rcalc.py".

Dovresti notare che ho creato quattro "mark" in esso (@mark.add, @mark.sub, @mark.multi, @mark.sub).

Un "mark" per ogni test. Dai un'occhiata anche al file "pytest.ini". Questo file è necessario per elencare tutti i miei "mark".

Se ti voglia testare il modulo, di solito eseguo il comando "pytest" senza flag. Come: pytest

Me questo eseguirà tutti i test nel file.

Immagina quanto tempo ci vorrebbe se avessi migliaia di test.

Per eseguire solo un test, o solo una serie di test, è meglio utilizzare un "mark".

Quindi, diciamo che voglio solo eseguire il test di addizione. Cosa devo fare?

Devo solo eseguire "pytest" con l'aggiunta di "mark".

Come questo: pytest -m add

Posso anche eseguire sole due tipo di test. Come add e sub.

Per questo devo eseguire "pytest" con un "OR" nel flag "mark".

Come questo: pytest -m "add or sub"

Puoi fare gli altri test da solo (multi e div).

E questo è tutto per oggi. Nel prossimo articolo ti mostrerò come utilizzare i "fixtures".
