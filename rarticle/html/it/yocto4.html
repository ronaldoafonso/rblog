Ma cos'è il Poy?

Poly è conosciuto come una distribuzione di riferimento del progetto Yocto.

È un template per costruire un sistema Linux personalizzato.

Contiene progetti come BitBake, OpenEmbedded Core e Metadata che ci guidano nel processo di creazione di un sistema Linux personalizzato.

Descriverò tutti questi altri progetti a turno, ma qui vediamo come iniziare con poky.

Prima di tutto, dobbiamo clonare il codice poky ed eseguirlo nella nostra build machine.

Fondamentalmente questo:

git clone http://git.yoctoproject.org/cgit/cgit.cgi/poky

cd pody

git fetch --tags

git checkout tags/yocto-3.2

Nota che sto usando un branch specifico qui, ok?

Userò il target "default" qui. È il modo più semplice per iniziare. Dopo aver ottenuto l'idea di base di come funziona Yocto, puoi provare quelli più avanzati.

source oe-init-build-env

bitbake core-image-minimal

Ora dovresti avere un'immagine che puoi eseguire sul tuo computer. Ovviamente lo eseguirai tramite QEMU, che è un emulatore.

runqemu qemux86

Dopo alcuni secondi dovresti vedere un dispositivo in esecuzione come se fosse un dispositivo incorporato.
