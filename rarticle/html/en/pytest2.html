Pytest - Article 2

This is the second article about the most used Python framework for testing, "pytest".

You can find the first one here.

In the first article I just explained how to install "pytest".

So, I'm assuming that you already have a virtual environment with "pytest" installed.

The topic of this article is the "mark" feature of "pytest".

Let me ask you something.

How could you execute just some of your tests instead of all the suite of tests?

Just in case you've forgotten it, you can find the "rcalc" project's code here

So, let's pretend that we have a Python module called "rcalc.py" and that we need to test four basic operations of it.

Usually we create a "test_rcalc.py" file to test the "rcalc.py" module.

Come on! Do it, clone my "rcalc" repository, check out the tag "v1.0" and take a look at the files. Specially the "test_rcalc.py" file.

You should notice that I created four "marks" in it (@mark.add, @mark.sub, @mark.multi, @mark.sub).

One "mark" for each test. Take a look at the "pytest.ini" file too. It's needed to list all my "marks".

If I'd like to test the module, I usually would execute the "pytest" command without flags. Like: pytest

But that would execute all the tests in the file.

Imagine how long it would take if I had thousands of tests.

In order to execute just one test, or just a set of tests, I'd better use a "mark".

So, let's say that I just want to execute the addition test. What do I have to do?

I just have to execute "pytest" with the "mark" add.

Like this: pytest -m add

I could also execute just two kinds of test. Like add and sub.

In order to do it, I should execute "pytest" with a "OR" in the "mark" switch.

Like this: pytest -m "add or sub"

You can do the other tests, multi and div, by yourself

And that's it for today. In the next article I'm goint to show you how to use "fixtures".
