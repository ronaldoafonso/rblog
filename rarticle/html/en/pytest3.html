Pytest - Article 3

This is the third article about the most used Python framework for testing, "pytest".

You can find the second one here.

Today I'm going to talk about "fixture".

A "fixture" is basically used to initialize your tests.

Maybe you are asking yourself, how come?

Let's take a look at our "rcalc" project.

If you clone it and use the "v2.0" tag you will see that I created the "conftest.py" file.

Inside that file you should see a "fixture", or a function, that simply returns a sequence of "tupes".

Those "tuples" are intented to serve as data for our tests.

I just need to define it in one place and can use it in any test.

Take a look at the "test_rcalc.py" file too.

There you could see that I just passed the "fixture" as parameter for my tests.

And inside the tests I can use my "data generator".

Now every time you execute your tests, "pytest" will run the "fixture" for you.

Another example of "fixtures" is database initialization.
