Now, it's time to talk about the poky project.

So, what is poky?

Poky is known as a reference distribution of the Yocto project.

That is, it's a template for building a custom Linux system.

It contains projects like BitBake, OpenEmbedded Core, and Metadata that guides us in the process of building a custom Linux system.

I'll describe all of these other projects in turn, but here let's see how to get started with poky.

First of all, we need to clone the poky code and execute it in our build machine.

Basically this:

git clone http://git.yoctoproject.org/cgit/cgit.cgi/poky

cd pody

git fetch --tags

git checkout tags/yocto-3.2

Note that I'm using a specific branch here, ok?

I'm going to use the "default" target here. It's the easiest way to get started. After you get the basic idea of how Yocto works, you can try more advanced ones.

source oe-init-build-env

bitbake core-image-minimal

Now you should have an image that you can run on your our computer. Of course, you're going to run it through QEMU, which is an emulator.

runqemu qemux86

After some seconds you should see a device running like it was an embedded device.

That's it.
