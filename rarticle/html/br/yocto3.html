Agora é hora de falar sobre como preparar uma build machine.

O que é ou para que serve uma build machine?

Para criar uma imagem customizado do seu sistema Linux, você irá precisar de outro sistema Linux.

Bem, você precisa de uma distribuição Linux "tradicional" onde possa baixar o projeto Yocto, e aí então construir a sua distribuição Linux.

Vou usar aqui o Debian 10 "buster", mas existem outras distribuições que podem ser usadas como uma build machine.

No presente momento, os pré-requisitos para uma build machine são:

  • 50 Gbytes de espaço em disco.
  • Git 1.8.3.1 ou mais atual.
  • tar 1.28 ou mais atual.
  • Python 3.50 ou mais atual.
  • gcc 5.0 ou mais atual.

Dependendo da sua distribuição Linux, você precisará de uma certa lista de pacotes instalados.

No caso do Debian 10, você precisará:

apt-get install gawk wget git-core diffstat unzip texinfo gcc-multilib build-essential chrpath socat cpio python3 python3-pip python3-pexpect xz-utils debianutils iputils-ping python3-git python3-jinja2 libegl1-mesa libsdl1.2-dev pylint3 xterm python3-subunit mesa-common-dev

É isso.
