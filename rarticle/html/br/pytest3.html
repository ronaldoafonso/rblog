Pytest - Artigo 3

Este é o terceiro artigo sobre o framework de testes mais usado em Python, o "pytest".

O segundo pode ser encontrado aqui

Hoje vou falar sobre "fixture".

Um "fixture" é usado basicamente para inicializar seus testes.

Talvez você esteja se perguntando, como assim?

Vamos dar uma olhada em nosso projeto rcalc.

Se você cloná-lo e usar a tag "v2.0", você verá que eu criei o arquivo "conftest.py".

Neste arquivo você verá um "fixture", ou uma função, que simplemente retorna uma sequência de "tupes".

Essas "tupes" serviram como dados para os nossos testes.

Eu só preciso declarar um "fixture" em um lugar e posso usá-lo em qualquer teste.

Dê uma olhada no arquivo "test_rcalc.py" também.

Lá você verá que eu passei o "fixture" como parâmetro para os meus testes.

Dentro dos testes eu posso usar o meu "data generator".

Agora toda vez que você executar seus tests, o "pytest" irá rodar o "fixture" para você.

Um uso muito comum dos "fixtures" é na inicialização de banco de dados.

Isso é tudo. Eu espero que estes artigos foram úteis para você.
